# Foundation Stack — Infrastructure Engineering Build Log

I've spent 5 years in cloud operations and change management on enterprise Splunk Cloud infrastructure — coordinating production changes, managing P1/P2 incident escalations, working daily alongside SRE and technical consulting teams. I understand how production infrastructure breaks and how it gets fixed. I haven't been the one building and administering it.

Everything here is something I built, broke, fixed, or figured out on a terminal.

No passive learning. No collecting courses.

**Lab environment:** Rocky Linux VM and Ubuntu on VMware Fusion, AWS Free Tier.

---

## Why Public

The people hiring for the roles I'm targeting will search my name. I want them to find someone who shows their work.

## The Session Logs

Every learning session gets committed. Each log contains the concept in plain language, exact commands run, actual output, what broke, and how it got fixed. Not curated to look impressive — documented to be honest and useful.
